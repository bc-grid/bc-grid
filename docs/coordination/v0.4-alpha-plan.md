# v0.4 Alpha Coordination Plan

Coordinator-owned plan for the `0.4-alpha` train. This release should prove editable ERP grid workflows before the project moves into broader spreadsheet-style paste/fill and later reporting features.

> **2026-05-02 coordination reset:** the 5-worker plan is retired. Claude coordinates from `~/work/bc-grid`. Active workers are worker1 Claude, worker2 Codex, worker3 Claude. Worker4 and worker5 are retired.

## Release Thesis

`v0.4-alpha` is the first editing-depth alpha:

- editable client grids are usable with built-in editors;
- server-backed edit grids have a clear contract for pending mutations and refetches;
- keyboard and accessibility behavior is pinned by tests;
- examples and docs make the features discoverable without tribal knowledge.

This is still an alpha. It does not need every spreadsheet workflow. User-facing Excel paste and fill handle stay in the `v0.6.0` range/clipboard milestone unless they are already complete and low risk.

## Coordinator Responsibilities

- Keep workers off Playwright, smoke-perf, and broad benchmarks.
- Review and merge worker PRs in the order that reduces conflict risk.
- Run type-check, focused unit tests, package builds, Playwright, and example-app visual checks.
- Keep `docs/queue.md` and `docs/coordination/release-milestone-roadmap.md` honest as tasks merge.
- When the gates below pass, recommend a package bump and publish.

## Worker Lanes

### Worker1 - Server Grid Stability and Server Edit Contracts (Claude)

- Stabilize server-backed sort/filter/search/group/page refresh so stale page rows do not client-sort or flicker while the server response is pending.
- Harden paged `BcServerGrid` contract tests for total rows, loaded page rows, stale responses, visible-column query payloads, and query-driven page reset.
- Add or refine server error/retry surfaces only if they fit the coordinator's loading/status polish.
- Docs for bsncraft-style customer grids: total rows, current page rows, reload policy, validation, rollback, conflict handling.

### Worker2 - Filters, Tool Panels, and v0.6 Prep (Codex)

- Polish filter popup keyboard contracts and filters panel active summaries without regressing the shadcn/Tailwind v4 chrome.
- Keep filter/sidebar/tool-panel work scoped and visual-quality-gated.
- Prepare `v0.6.0` range/clipboard helper work only after v0.4 filter/server/edit blockers are stable. (Renumbered from v0.5 after the 2026-05-02 audit-refactor pivot — current v0.5 is the audit-driven ergonomics refactor; see `docs/coordination/v0.5-audit-refactor-plan.md`.)
- Do not adjust bundle baselines to pass CI; the coordinator owns bundle policy.

### Worker3 - Editors, Validation, Lookup UX (Claude)

- F2, printable seed, Enter, Shift+Enter, Tab, Shift+Tab, Escape, async pending, validation rejection.
- ARIA and focus tests for editor lifecycle.
- Portal-level validation/error surfaces, built-in editor accessible names/descriptions, and lookup/select/autocomplete typed-value contracts.
- Custom editor guidance that matches built-in editor contracts.

## Coordinator-Owned Polish

## v0.4 Alpha Gates

- `bun run type-check`
- `bun test` or a justified focused equivalent while the merge train is moving
- `bun run build:packages`
- API surface and bundle-size checks
- Coordinator Playwright pass over examples covering editing, server edit, grouping discovery, pinned columns, filters, and tool panels
- Published metadata has no internal version skew
- `bsncraft` can install the candidate and type-check the representative grid wrapper

## Non-Goals

- Charts remain post-1.0.
- Full spreadsheet fill handle remains `v0.6.0`.
- Bug-for-bug AG Grid parity is not a goal; clean-room behavior comparison is allowed.
- Workers do not run Playwright, smoke-perf, or broad benchmarks.
